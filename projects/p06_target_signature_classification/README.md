# P06: Lightweight Target Signature Classification Example

P06 is a compact Python equivalent of MATLAB radar target-classification
examples.  It generates angle-dependent RCS-like target signatures from simple
point-scatterer geometry and compares a handcrafted-feature baseline with a
small 1-D CNN.

## MATLAB reference

- [Radar Target Classification Using Machine Learning and Deep Learning](https://www.mathworks.com/help/radar/ug/radar-target-classification-using-machine-learning-and-deep-learning.html)

The Python example keeps the same teaching idea—simple radar target return
classification with classical and neural models—but uses a lightweight
point-scatterer model that runs without external data.

## Task

Classify three target-signature families from aspect-varying complex returns:

`cylinder`, `cone`, `plate`

This is target signature classification, not SAR image despeckling and not DoA
mapping.  The input is a two-channel magnitude/phase sequence produced by a
small point-scatterer model.

## Interpretation

Each sample is an aspect-angle sweep of a lightweight monostatic scattering
model:

```text
target class -> point scatterer geometry -> aspect angle theta
             -> complex return s(theta)
             -> magnitude/phase sequence
```

The squared magnitude of the return can be read as an angle-dependent
RCS-like pattern.  P06 keeps phase as an additional channel, so the classifier
sees a complex target-signature sequence rather than only scalar RCS values.

## Python equivalent

```text
simple target geometry + aspect variation
    -> complex monostatic response + AWGN
    -> magnitude/phase signature
    -> handcrafted baseline or tiny 1-D CNN classifier
```

## Architecture

- **Model**: TinySignatureCNN
- **Input**: `(B, 2, 128)` — normalized magnitude and unwrapped phase channels
- **Output**: `(B, 3)` class logits
- **Loss**: CrossEntropyLoss with light label smoothing
- **Baseline**: handcrafted return descriptors + logistic regression/SVM

## Commands

```bash
# Fast CPU check
python train.py --generate --smoke

# Default lightweight run
python train.py --generate --epochs 10

# Eval only
python train.py --eval_only --checkpoint artifacts/best_model.pt

# Classical feature baseline
python evaluate_feature_baseline.py --data_dir data --model logreg
python evaluate_feature_baseline.py --data_dir data --model rbf_svm --max_train 1500

# Held-out aspect / low-SNR evaluation
python evaluate_generalization.py --generate --checkpoint artifacts/best_model.pt

# Example signatures
python plot_signatures.py
```

If a checkpoint was created with `--smoke`, add `--base_ch 8` to eval commands
that instantiate the neural model.

## Data

Generated by `generate_data.py` using `shared/target_signature.py`.

| Split | Filename | Default size |
|---|---|---:|
| Train | `data/signature_train.h5` | 3000 |
| Val | `data/signature_val.h5` | 600 |
| Test | `data/signature_test.h5` | 600 |

HDF5 schema includes `x (N,2,128)`, `y (N,)`, `features (N,F)`, `snr_db`, aspect
metadata, target class names, geometry/model metadata, and MATLAB reference URL.

## Metrics

| Metric | Description |
|---|---|
| accuracy | Overall class accuracy |
| f1_macro | Macro-averaged F1 across target classes |
| confusion_matrix | Which target signatures are confused |
| aspect-bin accuracy | Accuracy grouped by absolute aspect angle |
| SNR-bin accuracy | Accuracy grouped by SNR |

## Lightweight exploration knobs

- `--aspect_lo`, `--aspect_hi`: training/evaluation aspect coverage
- `--snr_lo`, `--snr_hi`: noise difficulty
- `--epochs`, `--lr`, `--base_ch`: learning and model capacity
- `evaluate_generalization.py --generate`: held-out aspect and low-SNR check
- `evaluate_feature_baseline.py --model`: classical baseline family
