# P01 Detection Contract — Verified Week 10 Technical Source

P01 is a controlled FMCW Range-Doppler Map (RDM) target-detection experiment.
It is not a full operational radar receiver. Week 10 lecture claims must be made
against this contract.

## Physics contract

- Radar: `FMCWRadar(fc=77e9, bw=1e9, T_chirp=50e-6, N_chirps=128, fs=10e6)` in `projects/p01_unet_detector/generate_data.py`.
- Signal model: beat frequency from two-way delay and Doppler phase over chirps (`shared/fmcw_simulator.py`).
- RDM: range FFT followed by Doppler FFT, using the positive range-frequency half.
- Neural input: 2 channels: noise-floor-referenced log magnitude and phase.
- Baseline input: schema-v2 datasets store `rdm_mag_linear` before display clipping so CFAR is evaluated on linear magnitude, not a reconstructed display image.

## Allowed simplifications

| Simplification | Why allowed for Week 10 | Not claimed |
|---|---|---|
| Amplitude is proportional to `sqrt(rcs)` | Focuses the lecture on RDM detection contracts and operating points | Full radar equation / calibrated received power |
| Mixed clutter is synthetic zero-Doppler/distributed/multipath scatterers | Gives controlled failure modes for CFAR and neural detectors | Site-specific environmental clutter |
| Target mask is a 5-pixel cross around quantized range/Doppler bin | Matches the main teaching label contract for Hann-windowed point targets | Semantic object segmentation mask |
| Single-receiver RDM is used for P01 detection | Keeps Week 10 focused on range/Doppler detection | Full MIMO angle processing |

## Dataset schema v2

Required HDF5 datasets:

- `x`: `(N, 2, 128, 250)` neural input.
- `y`: `(N, 1, 128, 250)` binary detection mask.
- `rdm_mag_linear`: `(N, 128, 250)` linear RDM magnitude for CFAR.
- `snr_db`, `n_targets`, `clutter_power_db`, `noise_floor`.
- `target_range_m`, `target_velocity_mps`, `target_rcs`, `target_range_bin`, `target_doppler_bin`.
- `range_axis_m`, `velocity_axis_mps`, `schema_version`.

## Baseline policy

CA-CFAR is the primary classical baseline. A valid CFAR result must report:

- guard cells,
- training cells,
- design `pfa`,
- split used for selecting the operating point,
- test split used for final reporting.

CFAR must be evaluated from `rdm_mag_linear`. Do not run CFAR from clipped or normalized display channels.

## Neural detector policy

A valid U-Net result must report:

- checkpoint path / training command,
- validation threshold-selection policy,
- selected threshold,
- test split metrics,
- whether the run is smoke-scale or full-scale.

The test split must not be used to choose thresholds.

## Metrics

Report at minimum:

- `Pd`: pixel-level recall on target mask pixels,
- `Pfa`: false detections over non-target pixels,
- `Precision`,
- `F1`,
- operating-point curves (`Pd` vs `Pfa`) for both CFAR and U-Net.

## Week 10 claim rule

A Week 10 result is valid only if it points to saved JSON artifacts generated by
`analyze_data_contract.py`, `evaluate_cfar.py`, `evaluate_unet.py`, and
`make_verified_figures.py`. Otherwise it must not be shown as a result.
