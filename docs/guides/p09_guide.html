1. 이 프로젝트는 뭘 하는 건가요?

Range-Doppler Map(RDM)이란?

FMCW 레이다는 송신 chirp과 수신 신호를 믹싱하여 Beat signal을 만들고, 이를 2D FFT(Range FFT + Doppler FFT) 처리하면 Range-Doppler Map(RDM)을 얻습니다. RDM의 가로축은 거리(range), 세로축은 속도(Doppler)를 나타내며, 각 셀의 값은 해당 거리·속도 조합에서의 반사 에너지입니다. 

Beat signal  →  Range FFT  →  Doppler FFT  →  RDM
(N_chirps × N_samples)          (N_chirps × N_range_bins)

RDM 축:
  세로(행): Doppler bins → 속도 분해능 = v_max / (N_chirps/2)
  가로(열): Range bins   → 거리 분해능 = c / (2 · BW)

표적: RDM에서 밝은 점(peak)으로 나타남

왜 RDM 해상도가 제한되는가?

RDM의 해상도는 하드웨어 파라미터에 의해 물리적으로 제한됩니다:

해상도 결정 요인 개선 방법 현실적 제약
거리 해상도 (range) 대역폭(BW) ↑ BW 늘리기 RF 부품 비용, 규제 대역
속도 해상도 (Doppler) chirp 수(N_chirps) ↑ 처프 더 쏘기 프레임 시간 증가, 실시간 처리 부담

예를 들어 이 프로젝트에서 HR RDM은 64×64입니다. chirp 수를 절반(32)으로 줄이거나, range sample을 절반으로 줄이면 RDM이 32×32로 작아집니다. Super-resolution은 이렇게 제한된 하드웨어 조건에서 수집한 저해상도(LR) RDM을 딥러닝으로 고해상도(HR)로 복원하는 기술입니다.

핵심 아이디어: 물리적 한계를 하드웨어가 아닌 소프트웨어(신경망)로 극복합니다. 표적이 RDM에서 어떤 패턴을 만드는지 대량의 LR/HR 쌍으로 학습하면, 새로운 LR 입력에서도 HR 구조를 복원할 수 있습니다.

3. 데이터 생성 코드 Walkthrough — generate_data.py

make_radar() — 레이다 파라미터 설정

def make_radar():
    return FMCWRadar(
        fc=77e9,          # 반송파 주파수: 77 GHz (자동차 레이다 대역)
        bw=1e9,           # 스윕 대역폭: 1 GHz → 거리 해상도 ≈ 0.15 m
        T_chirp=50e-6,    # 처프 주기: 50 μs
        N_chirps=64,      # 처프 수 = HR Doppler axis 크기 (64)
        fs=10e6,          # 샘플링 주파수 → Range samples 결정
        N_rx=1,           # 수신 안테나 수: 1
    )

N_chirps=64: Doppler FFT 크기가 64이므로 HR RDM의 세로 크기가 64가 됩니다. 이 값을 32로 줄이면 Doppler 해상도가 절반으로 떨어지는 저해상도 상황을 시뮬레이션할 수 있습니다. 이 프로젝트에서는 HR을 만들고 소프트웨어적으로 다운샘플링하여 LR을 시뮬레이션합니다.

rdm_to_db_normalized() — dB 변환 및 정규화

DB_MIN = -60.0   # dB 최솟값 클리핑
DB_MAX =   0.0   # dB 최댓값 (최대값 기준 0 dB)

def rdm_to_db_normalized(rdm_complex):
    mag = np.abs(rdm_complex)                        # 복소 RDM → 진폭

    # 진폭 → dB (최대값 기준 정규화)
    db = 20.0 * np.log10(mag / (mag.max() + 1e-30) + 1e-30)

    # [-60, 0] dB 범위로 클리핑
    db = np.clip(db, DB_MIN, DB_MAX)

    # [-1, 1] 범위로 선형 정규화
    db_norm = (db - DB_MIN) / (DB_MAX - DB_MIN) * 2.0 - 1.0
    return db_norm.astype(np.float32)

왜 dB 스케일을 사용하나?

RDM 진폭 값의 동적 범위(dynamic range)는 수십에서 수백만까지 매우 넓습니다. dB 스케일로 변환하면 이 범위가 압축되어 신경망이 강한 표적과 약한 표적을 동시에 학습하기 쉬워집니다. 클리핑 범위 [-60, 0] dB는 잡음 수준(-60 dB) 아래를 버리고 최대 신호(0 dB) 기준으로 정규화합니다.

[-1, 1] 정규화 이유

신경망 입력을 [-1, 1] 범위로 맞추면 초기 가중치와의 스케일이 맞아 학습 초기에 gradient가 적절하게 흐릅니다. 출력도 같은 범위이므로 L1 손실의 절대값이 일관된 스케일을 유지합니다.

generate_one_sample() — LR/HR 쌍 생성

def generate_one_sample(radar, rng, sample_seed):
    # 1. 표적 파라미터 랜덤 샘플링
    n_tgt = int(rng.integers(1, 5))       # 1~4 표적
    snr_db = float(rng.uniform(5.0, 25.0))# SNR 범위

    targets = [{'range': ..., 'velocity': ..., 'rcs': ...}
               for _ in range(n_tgt)]

    # 2. FMCW 시뮬레이션 → HR RDM (복소, 64×64)
    signal = generate_scene(radar, targets, snr_db=snr_db, seed=sample_seed)
    rdm = range_doppler_map(signal, window_range='hann', window_doppler='hann')
    rdm_hr = rdm[0, :, :HR_RANGE]         # (64, 64) 복소 행렬

    # 3. HR → dB 정규화 → (64, 64) float
    y_map = rdm_to_db_normalized(rdm_hr)

    # 4. 2× 다운샘플링으로 LR 생성 (average pooling 효과)
    x_map_lr = zoom(y_map, 0.5, order=1)  # (32, 32): bilinear downsampling

    # 5. Peak mask: 표적 위치 3×3 영역 표시 (HR 공간 기준)
    peak_mask = np.zeros((64, 64), dtype=np.float32)
    for tgt in targets:
        r_bin = int(round(tgt['range'] / range_res))
        v_bin = int(np.argmin(np.abs(vel_axis - tgt['velocity'])))
        # 3×3 이웃 영역 marking
        for di in (-1, 0, 1):
            for dj in (-1, 0, 1):
                peak_mask[clip(v_bin+di), clip(r_bin+dj)] = 1.0

    return x_map_lr[np.newaxis], y_map[np.newaxis], peak_mask[np.newaxis]
    #      (1,32,32)              (1,64,64)           (1,64,64)

zoom(y_map, 0.5, order=1)는 scipy의 bilinear 다운샘플링입니다. 각 LR 픽셀이 2×2 HR 픽셀의 평균에 해당하므로, 물리적으로 "chirp 수 절반"과 유사한 상황을 만듭니다. peak_mask는 표적 위치 평가(Peak Location Error)에만 사용되며, 학습에는 사용하지 않습니다.

"이 파라미터를 바꾸면?" 실험 제안
  • n_tgt = rng.integers(1, 5): 표적 수 범위를 (3, 6)으로 바꾸면 RDM이 더 복잡해집니다. 표적이 많을수록 복원이 어려워지나요?
  • snr_db = rng.uniform(5.0, 25.0): 저SNR(0.0, 10.0)만 학습하면 고SNR 복원 성능에 어떤 영향이 있나요?
  • zoom(y_map, 0.5, order=1): order=0(nearest neighbor)으로 바꾸면 더 거친 LR을 만들어 복원이 어려워집니다. 성능이 얼마나 달라지나요?

4. 모델 코드 Walkthrough — model.py

ResidualBlock — Residual 연결의 역할

class ResidualBlock(nn.Module):
    """Conv-BN-ReLU-Conv-BN with skip connection."""

    def __init__(self, channels=32):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv2d(channels, channels, kernel_size=3, padding=1, bias=False),
            nn.BatchNorm2d(channels),
            nn.ReLU(inplace=True),
            nn.Conv2d(channels, channels, kernel_size=3, padding=1, bias=False),
            nn.BatchNorm2d(channels),   # 두 번째 Conv 뒤: activation 없음
        )
        self.relu = nn.ReLU(inplace=True)

    def forward(self, x):
        return self.relu(x + self.block(x))  # 입력 x + 변환 결과를 더함

Skip Connection이란?

x + self.block(x)에서 x가 바로 더해지는 경로가 skip connection입니다. 이 경로 덕분에 블록이 학습해야 하는 것은 "원본에서 얼마나 달라야 하는가"(잔차, residual)입니다. 잔차가 0에 가까우면 블록이 항등 함수(identity)처럼 동작하여, 불필요한 블록이 학습을 방해하지 않습니다.

Gradient Flow 개선

역전파(backpropagation) 시 skip connection이 gradient를 직접 앞 레이어로 전달합니다. 이 경로가 없으면 깊은 네트워크에서 gradient가 소실(vanishing gradient)되기 쉽습니다. 4개의 블록을 쌓아도 학습이 안정적인 이유가 바로 이 skip connection 덕분입니다.

SRResNetLite 전체 구조

class SRResNetLite(nn.Module):
    def __init__(self, n_res_blocks=4, channels=32, upscale=2):
        super().__init__()

        # 1. 초기 피처 추출
        self.head = nn.Sequential(
            nn.Conv2d(1, channels, kernel_size=3, padding=1),  # (B,1,32,32) → (B,32,32,32)
            nn.ReLU(inplace=True),
        )

        # 2. Residual 블록 4개 (공간 크기 유지: 32×32)
        self.res_blocks = nn.Sequential(
            *[ResidualBlock(channels) for _ in range(n_res_blocks)]
        )

        # 3. Post-residual conv (activation 없음 — PixelShuffle 전 안정화)
        self.post_res = nn.Conv2d(channels, channels, 3, padding=1, bias=False)

        # 4. Upsample: PixelShuffle로 2× 업스케일
        self.upsample = nn.Sequential(
            nn.Conv2d(channels, channels * upscale * upscale, 3, padding=1),
            # channels → channels * 4 = 128 채널
            nn.PixelShuffle(upscale),   # (B,128,32,32) → (B,32,64,64)
        )

        # 5. 출력 conv
        self.tail = nn.Conv2d(channels, 1, kernel_size=3, padding=1)
        #                                  → (B,1,64,64)

    def forward(self, x):
        feat = self.head(x)                     # (B, 32, 32, 32)
        res  = self.res_blocks(feat)             # (B, 32, 32, 32)
        res  = self.post_res(res) + feat         # skip: res블록 전체를 건너뜀
        up   = self.upsample(res)                # (B, 32, 64, 64)
        out  = self.tail(up)                     # (B, 1, 64, 64)
        return out

PixelShuffle — 업스케일 원리

PixelShuffle(upscale=2) 동작:

입력: (B, C×r², H, W)     r = upscale = 2
출력: (B, C, r·H, r·W)

구체적 예:
  Conv2d(32→128, 3×3) → (B, 128, 32, 32)  ← 채널을 4배로 늘림
  PixelShuffle(2)      → (B,  32, 64, 64)  ← 채널 4개를 2×2 픽셀로 재배치

왜 PixelShuffle인가?
  Bilinear Upsample + Conv: 단순 보간 후 학습 → 블러 발생 우려
  Transposed Conv: checkerboard artifact 문제
  PixelShuffle: 채널에서 공간으로 정보를 재배치 → 더 선명한 출력

그림 4. PixelShuffle 업스케일 원리. 채널의 정보를 공간 해상도로 변환합니다.

Conv로 채널을 먼저 늘리는 이유: 신경망이 업스케일 전 충분한 정보를 채널에 담아두고, PixelShuffle이 이를 공간적으로 펼쳐 해상도를 높입니다. 이 방식은 학습 가능한 업스케일이므로, 단순 bilinear 보간보다 표적의 선명한 peak를 복원할 수 있습니다.

SRLoss — L1 + Sobel Gradient 손실 설계

class GradientLoss(nn.Module):
    """Sobel gradient loss: 경계(edge)의 선명도를 보존합니다."""
    def __init__(self):
        super().__init__()
        # Sobel x 방향 필터 (수직 경계 검출)
        kx = torch.tensor([[-1, 0, 1], [-2, 0, 2], [-1, 0, 1]], dtype=torch.float32)
        # Sobel y 방향 필터 (수평 경계 검출)
        ky = torch.tensor([[-1,-2,-1], [ 0, 0, 0], [ 1, 2, 1]], dtype=torch.float32)
        self.register_buffer("kx", kx.view(1, 1, 3, 3))
        self.register_buffer("ky", ky.view(1, 1, 3, 3))

    def forward(self, pred, target):
        gx_pred = F.conv2d(pred,   self.kx, padding=1)   # 예측의 x방향 기울기
        gy_pred = F.conv2d(pred,   self.ky, padding=1)   # 예측의 y방향 기울기
        gx_tgt  = F.conv2d(target, self.kx, padding=1)   # 정답의 x방향 기울기
        gy_tgt  = F.conv2d(target, self.ky, padding=1)   # 정답의 y방향 기울기
        return F.l1_loss(gx_pred, gx_tgt) + F.l1_loss(gy_pred, gy_tgt)


class SRLoss(nn.Module):
    """L1 + weighted gradient loss."""
    def forward(self, pred, target):
        return self.l1(pred, target) + self.grad_weight * self.grad(pred, target)
        #       픽셀 수준 오차      +  0.1 × 경계 선명도 오차

L1 Loss만으로 충분하지 않은 이유

L1 Loss는 픽셀별 절대 오차를 최소화합니다. 하지만 RDM에서 중요한 것은 표적의 선명한 peak입니다. L1만 쓰면 모델이 전체 오차 합을 줄이기 위해 peak를 흐리게(blur) 복원하는 경향이 생깁니다. 경계가 흐려도 평균 픽셀 오차는 작을 수 있기 때문입니다.

Sobel Gradient Loss의 역할

Sobel 필터는 이미지의 급격한 변화(경계)를 강조합니다. RDM에서 표적은 주변보다 훨씬 밝은 peak로 나타나므로 강한 gradient를 가집니다. 예측의 gradient가 정답의 gradient와 일치하도록 추가 손실을 부여하면, 모델이 peak의 위치와 선명도를 더 잘 보존합니다. grad_weight=0.1은 L1 대비 10% 가중치로 경계 손실을 추가합니다.

평가 지표 설명

지표 수식 의미 목표값
PSNR (dB) 10·log10(MAX² / MSE) 픽셀 수준 재구성 품질. 높을수록 원본에 가까움. 영상처리 표준 지표. > 33 dB
NMSE ‖pred-target‖² / ‖target‖² 정규화 평균 제곱 오차. dB로 표현 시 낮을수록 좋음. < -28 dB
Peak Location Error (px) GT peak와 예측 peak 간 평균 픽셀 거리 표적 위치 정확도. 낮을수록 표적을 정확한 위치에 복원함. < 0.8 px

PSNR은 전체적인 화질을 나타내지만, 레이다에서는 표적 위치 정확도가 더 중요할 수 있습니다. 따라서 Peak Location Error를 함께 확인하세요. PSNR이 높아도 표적 위치가 어긋나면 실용적 가치가 낮습니다.

peak_localization_error() — 표적 위치 오차 측정

def peak_localization_error(pred_np, peak_mask_np):
    # peak_mask: GT 표적 위치 (3×3 영역 = 1)
    # pred_np  : 복원된 HR map (B, 1, 64, 64)

    for i in range(N):
        # GT peak 중심 좌표 계산 (connected components)
        labeled, n_obj = label(mask > 0.5)
        gt_centers = center_of_mass(mask, labeled, range(1, n_obj+1))  # 각 표적 무게중심

        # 예측 map에서 상위 k개 픽셀 위치 추출
        flat_idx = np.argsort(pred_map.ravel())[::-1][:n_obj * 3]
        pred_centers = [(idx // W, idx % W) for idx in flat_idx]

        # 각 GT 표적 → 가장 가까운 예측 peak 거리
        D = cdist(gt_centers, pred_centers)   # 유클리드 거리 행렬
        matched_dist = D.min(axis=1)          # 각 GT에 가장 가까운 예측
        errors.extend(matched_dist)

GT 표적 수만큼 예측 peak을 찾아 가장 가까운 쌍을 매칭합니다. Bicubic은 표적 주변을 흐리게 복원하므로 peak가 번지고, Peak Location Error가 큽니다. SRResNetLite는 Sobel 손실 덕에 더 날카로운 peak를 복원합니다.
