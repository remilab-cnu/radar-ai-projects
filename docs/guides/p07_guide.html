1. 이 프로젝트는 뭘 하는 건가요?

전이중(Full-Duplex) 통신이란?

일반적인 무선 통신은 반이중(Half-Duplex)으로 동작합니다. 즉, 송신과 수신이 다른 시간 슬롯(TDD) 또는 다른 주파수(FDD)를 사용합니다. 전이중(Full-Duplex)은 같은 시간, 같은 주파수에서 동시에 송신과 수신을 수행합니다. 이론적으로 스펙트럼 효율이 2배가 되지만, 큰 문제가 있습니다.

자기 간섭(Self-Interference, SI) 문제: 송신 안테나에서 나온 신호가 같은 장치의 수신 안테나로 직접 들어옵니다. TX 출력은 수 W(와트) 수준이지만 표적 에코는 수 nW(나노와트)입니다. 데이터 생성 코드의 sir_db는 SI-to-echo power ratio(사실상 ISR)로, −10 ~ +20 dB 범위에서 SI가 표적 에코의 0.1~100배까지 변해서 수신기가 표적을 아예 볼 수 없게 됩니다.

자기 간섭 채널 모델

수신 신호는 세 성분의 합입니다:

rx_mix(t) = SI_channel(tx(t))  +  target_echo(t)  +  noise(t)
            ────────────────────   ─────────────────
                  y_si                 y_clean

SI 채널 모델:
  y_si = h * tx  +  α₃ · tx · |tx|²    ← FIR 컨볼루션 + 3차 비선형항

  h : 복소 FIR 계수 (2~5탭, 지수감쇠 포락선)
  α₃: 비선형 계수 (30% 확률로 추가)

목표:
  SI 추정값 si_hat ≈ y_si  →  clean_hat = rx_mix - si_hat ≈ y_clean

그림 1. Full-Duplex 수신 신호 모델

SI 채널은 단순한 선형 FIR 필터가 아닙니다. PA(전력증폭기)와 믹서의 비선형성으로 인해 3차 비선형항이 추가되는 경우가 있습니다. 이것이 선형 적응 필터(NLMS)의 한계이고, 딥러닝이 필요한 핵심 이유입니다.

3. 데이터 생성 코드 Walkthrough — generate_data.py

generate_chirp() — TX 기준 신호 생성

def generate_chirp(N, rng):
    t = np.arange(N)               # 시간 인덱스 0, 1, ..., 511
    f0 = rng.uniform(0.0, 0.1)    # 시작 주파수 (정규화)
    f1 = rng.uniform(0.3, 0.45)   # 끝 주파수 (정규화)
    k  = (f1 - f0) / N            # chirp rate

    phase0 = rng.uniform(0, 2*π)  # 무작위 초기 위상

    # FMCW chirp: 주파수가 선형으로 증가하는 복소 지수 함수
    chirp = exp(j * (2π * (f0*t + 0.5*k*t²) + phase0))
    return chirp   # (512,) complex

모델에는 이 chirp의 실수부와 허수부가 각각 ch0, ch1로 들어갑니다. 초기 위상과 주파수를 무작위로 변화시키는 이유는 모델이 특정 chirp 파형에 과적합되지 않도록 하기 위해서입니다.

apply_si_channel() — SI 채널 적용

def apply_si_channel(tx, rng, n_taps_range=(2, 5), nonlinear=False):
    n_taps = rng.integers(2, 6)      # 탭 수: 2~5 (무작위)
    decay = rng.uniform(0.3, 0.7)   # 지수 감쇠 계수

    # 복소 FIR 계수 (지수감쇠 포락선 → 가까운 경로가 더 강함)
    h_real = rng.standard_normal(n_taps) * decay**np.arange(n_taps)
    h_imag = rng.standard_normal(n_taps) * decay**np.arange(n_taps)
    h = (h_real + 1j*h_imag) / sqrt(sum(|h|²))   # 전력 정규화

    # FIR 컨볼루션 (같은 길이 유지)
    y_si = h[0]*tx + h[1]*tx[shift(1)] + ...

    # 3차 비선형항 (30% 확률로 추가)
    if nonlinear:
        alpha3 = rng.uniform(0.05, 0.2)
        y_si += alpha3 * tx * |tx|²   ← 비선형 SI 성분

    return y_si, h
왜 비선형항이 문제인가? 선형 FIR 부분은 NLMS 같은 적응 필터로 추정할 수 있습니다. 그러나 tx · |tx|² 항은 TX 신호의 비선형 함수이므로 선형 필터로는 완벽히 제거할 수 없습니다. 30% 샘플에 비선형 SI가 포함되어, 모델은 선형+비선형 SI를 모두 처리해야 합니다.

make_sample() — 하나의 학습 샘플 조립

def make_sample(rng, sir_db, snr_db):
    tx    = generate_chirp(512, rng)           # TX 기준 신호
    y_si, _ = apply_si_channel(tx, rng,
                  nonlinear=(rng.random() < 0.3))  # SI 신호

    echo  = generate_target_echo(512, rng)    # 교육용 단순 표적 에코(지연된 협대역 tone)

    # 저장된 sir_db는 SI-to-echo power ratio(dB)에 맞게 echo 전력 조정
    # ratio = P_si / P_echo → echo 스케일 = sqrt(P_si / (ratio_lin * P_echo))
    scale_echo = sqrt(P_si / (SIR_lin * P_echo))
    echo = echo * scale_echo

    # 잡음 (SNR = P_si / P_noise 기준)
    noise = noise_std * (randn(512) + j*randn(512))

    y_clean = echo + noise    # 표적 에코 surrogate + 잡음 (복원 목표)
    rx_mix  = y_si + y_clean  # 실제 수신 신호 (SI + 표적 + 잡음)

    return {
        "tx_ref":  [real(tx),     imag(tx)],     # (2, 512)
        "rx_mix":  [real(rx_mix), imag(rx_mix)], # (2, 512)
        "y_si":    [real(y_si),   imag(y_si)],   # (2, 512) — SI 추정 목표
        "y_clean": [real(y_clean),imag(y_clean)],# (2, 512) — clean 복원 목표
    }

이 프로젝트의 sir_db는 관례적인 SIR(신호/간섭)이 아니라 SI-to-echo ratio입니다. sir_range=(-10, +20 dB)이므로 값이 클수록 어렵고, +20 dB에서는 SI가 표적 에코보다 100배 강합니다.

"이 파라미터를 바꾸면?" 실험 제안
  • nonlinear 확률 0.3 → 0.0: 비선형 SI 없이 학습하면 NLMS와 격차가 줄어드나요?
  • sir_range=(-10, 20) → (0, 20): SI가 항상 표적보다 강한 어려운 환경에서 성능은?
  • n_taps_range=(2, 5) → (10, 20): 더 긴 SI 채널 응답에서 U-Net이 충분한가요?

4. 모델 코드 Walkthrough — model.py

U-Net 구조란?

U-Net은 원래 의료 영상 분할을 위해 제안된 구조로, Encoder(다운샘플링)와 Decoder(업샘플링)를 Skip Connection으로 연결합니다. 이 프로젝트는 1D 시계열 신호에 U-Net을 적용합니다. 

입력 (B, 4, 512)
  │
  ├─ enc1: Conv1d(4→32, k=7) ─────────────────────────────────────┐
  │         (B, 32, 512)                                 skip s1   │
  │                                                                 │
  ├─ enc2: Conv1d(32→64, k=3, stride=2) ──────────────────────┐   │
  │         (B, 64, 256)                             skip s2   │   │
  │                                                             │   │
  ├─ enc3: Conv1d(64→128, k=3, stride=2)                       │   │
  │         (B, 128, 128)                                       │   │
  │                                                             │   │
  ├─ bottleneck: Conv(128→256→128)                              │   │
  │         (B, 128, 128)                                       │   │
  │                                                             │   │
  ├─ dec2: ConvTranspose1d(128→64) + cat(s2) ──────────────────┘   │
  │         (B, 64, 256)                                            │
  │                                                                 │
  ├─ dec1: ConvTranspose1d(64→32) + cat(s1) ───────────────────────┘
  │         (B, 32, 512)
  │
  └─ head: Conv1d(32→2, k=1)
            (B, 2, 512)   ← SI 추정값

그림 4. SICUNet 1D U-Net 구조와 Skip Connection

SICUNet 클래스 코드

class SICUNet(nn.Module):
    def __init__(self):
        super().__init__()
        # Encoder (점점 채널 늘리고 길이 줄임)
        self.enc1 = ConvBlock1d(4,  32,  kernel_size=7, padding=3)   # (B,32,512)
        self.enc2 = ConvBlock1d(32, 64,  kernel_size=3, stride=2, padding=1)  # (B,64,256)
        self.enc3 = ConvBlock1d(64, 128, kernel_size=3, stride=2, padding=1)  # (B,128,128)

        # Bottleneck (가장 압축된 표현)
        self.bottleneck = nn.Sequential(
            ConvBlock1d(128, 256, kernel_size=3, padding=1),
            ConvBlock1d(256, 128, kernel_size=3, padding=1),
        )

        # Decoder (업샘플링 + skip connection으로 세부정보 복원)
        self.dec2 = UpBlock1d(in_ch=128, skip_ch=64,  out_ch=64)   # (B,64,256)
        self.dec1 = UpBlock1d(in_ch=64,  skip_ch=32,  out_ch=32)   # (B,32,512)

        # 출력 헤드 (1×1 Conv로 채널만 변환)
        self.head = nn.Conv1d(32, 2, kernel_size=1)                 # (B,2,512)

    def forward(self, x):   # x: (B, 4, 512)
        s1 = self.enc1(x)       # (B, 32, 512) — skip 저장
        s2 = self.enc2(s1)      # (B, 64, 256) — skip 저장
        s3 = self.enc3(s2)      # (B, 128, 128)

        b  = self.bottleneck(s3) # (B, 128, 128)

        d2 = self.dec2(b, s2)   # (B, 64, 256)  ← s2 skip 사용
        d1 = self.dec1(d2, s1)  # (B, 32, 512)  ← s1 skip 사용

        return self.head(d1)    # (B, 2, 512)

Skip Connection의 역할

Encoder가 다운샘플링을 거치면 시간 해상도가 낮아집니다. SI는 시간 정밀도가 중요한 신호이므로, Decoder만으로 세밀한 파형을 복원하기 어렵습니다. Skip Connection은 각 Encoder 단계의 고해상도 특징을 Decoder에 그대로 전달합니다. 이로써 Decoder는 저해상도의 전체 컨텍스트와 고해상도의 세부 정보를 동시에 활용합니다.

UpBlock1d — 업샘플링 블록

class UpBlock1d(nn.Module):
    def forward(self, x, skip):
        x = self.up(x)  # ConvTranspose1d로 길이 2배
        # 길이 불일치 보정
        if x.shape[-1] != skip.shape[-1]:
            x = x[..., :skip.shape[-1]]
        x = torch.cat([x, skip], dim=1)  # 채널 방향으로 합치기
        return self.conv(x)

ConvTranspose1d는 Convolution의 역연산으로 길이를 2배로 늘립니다. 길이가 정확히 맞지 않을 수 있어 잘라내는 처리가 포함됩니다. 이후 skip 특징과 채널 방향으로 합쳐서 Conv로 처리합니다.

그림 5. main() 실행 흐름

SICLoss — Dual Loss 설계 근거

class SICLoss(nn.Module):
    def forward(self, si_hat, y):
        # y: (B, 4, 512) = [y_si(2ch), y_clean(2ch)]
        y_si    = y[:, :2, :]    # SI 정답 신호
        y_clean = y[:, 2:, :]   # 클린 신호 정답

        # SI 추정값으로부터 클린 신호 복원
        rx_mix    = y_si + y_clean         # rx_mix = SI + clean
        clean_hat = rx_mix - si_hat        # 잔차가 복원된 클린 신호

        loss_si    = SmoothL1(si_hat, y_si)       # SI 추정 손실
        loss_clean = SmoothL1(clean_hat, y_clean) # 클린 복원 손실

        return 0.7 * loss_si + 0.3 * loss_clean  # 가중 합

SI 추정 손실 (가중치 0.7)

모델이 SI를 정확하게 추정하도록 강제합니다. 가중치가 더 높은 이유는, SI 추정이 정확해야 클린 신호 복원도 자연히 좋아지기 때문입니다. 이 항이 없으면 모델이 SI를 무시하고 클린 신호를 0으로 예측하는 것이 손실을 줄일 수 있습니다.

클린 복원 손실 (가중치 0.3)

clean_hat = rx_mix - si_hat의 품질을 직접 최적화합니다. SI 추정이 약간 틀려도 "표적 신호를 잘 보존하는 방향"으로 보완합니다. 이 항이 없으면 SI를 과추정(overshoot)하여 표적 신호까지 지워버리는 현상이 생길 수 있습니다.

nlms_baseline() — NLMS 적응 필터 기준선

def nlms_baseline(tx_ref, rx_mix, n_taps=32, mu=0.05):
    w = zeros(32, complex)   # 필터 계수 초기화 (32탭 복소 FIR)

    for n in range(32, 512):
        x_buf = tx_ref[n:n-32:-1]    # 최근 32샘플 버퍼 (역순)
        y_hat = dot(w*, x_buf)        # SI 추정: 필터 출력

        si_hat[n] = y_hat
        e = rx_mix[n] - y_hat         # 오차 = 수신 - 추정

        # NLMS 계수 업데이트
        denom = dot(x_buf*, x_buf).real + 1e-10
        w += (mu / denom) * conj(e) * x_buf   # 오차 최소화 방향

    return si_hat
NLMS가 비선형 SI를 제거하지 못하는 이유: NLMS는 si_hat = w^H · x 형태의 선형 모델을 가정합니다. SI가 tx · |tx|²처럼 비선형이면, 선형 조합으로는 표현 자체가 불가능합니다. 32탭 필터를 더 길게 해도 근본적으로 해결되지 않습니다. 딥러닝 모델은 비선형 활성화 함수(ReLU)를 통해 이 비선형 관계를 학습합니다.

load_dataset() — 데이터 로드 방식

def load_dataset(split):
    data = load_hdf5(f"data/{split}.h5",
                     ["tx_ref", "rx_mix", "y_si", "y_clean"])

    # TX와 RX를 채널 방향으로 합쳐 4채널 입력 구성
    x = np.concatenate([data["tx_ref"], data["rx_mix"]], axis=1)
    #   data["tx_ref"]: (N, 2, 512)
    #   data["rx_mix"]: (N, 2, 512)
    #   x:             (N, 4, 512)  ← 모델 입력

    # SI와 clean을 채널로 합쳐 손실 함수 전달 편의상 묶음
    y = cat([y_si, y_clean], dim=1)  # (N, 4, 512)

    return TensorDataset(x, y)

TX 기준 신호(tx_ref)를 모델에 직접 제공하는 이유는, 모델이 "이 수신 신호에서 TX 성분이 어디에 있는가"를 참조할 수 있게 하기 위해서입니다. TX 없이 rx_mix만 입력으로 주면, 모델이 SI와 표적을 구분하기 훨씬 어렵습니다.

평가 지표 설명

지표 수식 의미 목표값
Cancellation Depth (dB) 10·log10(‖y_si‖² / ‖y_si − si_hat‖²) SI를 얼마나 잘 추정했는가. 클수록 좋음. > 35 dB
Output SIR Gain (dB) SIR_output − SIR_input SI 제거 후 표적/잔류SI 비율 개선량. 클수록 좋음. > 25 dB
Clean NMSE (dB) 10·log10(‖y_clean − clean_hat‖² / ‖y_clean‖²) 복원된 클린 신호의 정확도. 낮을수록 좋음. < −15 dB
Cancellation Depth 직관: 35 dB는 SI 신호 전력의 99.97%를 제거했다는 의미입니다. 20 dB는 99%, 40 dB는 99.99%입니다. 실제 레이다/통신 시스템에서는 40~60 dB 이상의 제거가 요구되므로, 이 프로젝트의 35 dB 목표는 DNN 학습의 의미 있는 첫 단계입니다.
