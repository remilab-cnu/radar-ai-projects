1. 이 프로젝트는 뭘 하는 건가요?

ULA 수신기와 빔포밍 기초

ULA(Uniform Linear Array)는 안테나를 일정 간격(λ/2)으로 나열한 배열 안테나입니다. N개의 안테나가 동일한 신호를 수신할 때, 각 안테나에 도달하는 신호의 위상(phase)이 입사각 θ에 따라 달라집니다. 이 위상 차이를 이용하면 특정 방향의 신호를 강조하거나 억압하는 빔포밍(beamforming)이 가능합니다.

N개 안테나에서 수신된 신호 벡터 x(t) ∈ ℂ^N은 다음과 같이 모델링됩니다: 

x(t) = a(θ_s)·s(t)  +  Σ a(θ_j)·j_k(t)  +  n(t)
       ────────────     ─────────────────────────
        원하는 신호         재머 신호 (1~2개) + 잡음

a(θ) = [1,  e^{j·π·sin(θ)},  e^{j·2π·sin(θ)},  ...,  e^{j·(N-1)π·sin(θ)}]^T
       ─────────────────────────────────────────────────────────────────────
                    스티어링 벡터 (steering vector) : N×1 복소 벡터

그림 1. ULA 신호 모델. a(θ)는 방향 θ에서 오는 신호의 안테나별 위상 지연을 나타냅니다.

스티어링 벡터 a(θ)의 k번째 원소 e^{j·k·π·sin(θ)}는, 안테나 간격이 λ/2일 때 k번째 안테나와 0번째 안테나 사이의 위상 차이를 나타냅니다. sin(θ)가 들어가는 이유는 안테나 간격 방향 성분만 위상에 기여하기 때문입니다.

공분산 행렬(Covariance Matrix)이란?

수신 신호 x(t)를 T개의 시간 샘플(스냅샷)에 걸쳐 측정하면, 샘플 공분산 행렬 R을 추정할 수 있습니다: 

R = (1/T) · X · X^H       ← X = [x(1), x(2), ..., x(T)] : N×T 행렬
                                   X^H : X의 켤레전치(Hermitian transpose)

결과: R ∈ ℂ^{N×N}  (이 프로젝트에서 N=8)

R의 물리적 의미:
  R[i,j] = 안테나 i와 안테나 j 수신 신호의 상관(correlation)
  대각 원소 R[i,i] = 안테나 i의 수신 전력
  재머가 강하면 → 재머 방향에 해당하는 성분이 R에 두드러짐

그림 2. 샘플 공분산 행렬의 정의와 물리적 의미

공분산 행렬은 배열 신호처리에서 매우 중요한 정보를 담습니다. 재머처럼 전력이 강한 신호는 R에 그 방향 정보를 강하게 남기기 때문에, R을 분석하면 재머 방향(DoA: Direction of Arrival)을 추정할 수 있습니다.

재머 문제 — 왜 null steering이 필요한가?

재머(Jammer)는 레이다나 통신 수신기를 방해하기 위해 의도적으로 강한 신호를 송출하는 간섭원입니다. JNR(Jammer-to-Noise Ratio)이 10~40 dB에 달하면, 원하는 신호가 재머에 완전히 묻힙니다.

핵심 문제: 수신기가 원하는 방향(θ_s)을 향해 빔을 형성해도, 재머가 다른 방향(θ_j)에서 매우 강하게 들어오면 SNR이 심각하게 저하됩니다. 해결책은 빔 패턴에 null(zero-response 방향)을 재머 방향에 배치하는 것입니다.

이 프로젝트의 목표: CovNet이 공분산 행렬 R에서 재머 방향 θ_j를 추정하고, 그 방향에 LCMV null을 배치하여 재머 억압 성능을 측정합니다. 재머가 2개인 장면에서는 더 강한(primary) 재머 하나만 레이블과 평가 대상으로 삼습니다.

Physics contract / Allowed simplification / Not claimed

  • Physics contract: λ/2 간격 8소자 ULA, narrowband far-field steering vector, snapshot covariance, 1~2 Gaussian jammer, 원하는 신호 look angle, LCMV null steering으로 구성됩니다. 평가는 예측 각도에 null을 둔 뒤 실제 재머 각도의 beam response를 측정합니다.
  • Allowed simplification: primary jammer 하나만 회귀/평가 대상으로 삼고, mutual coupling, array calibration error, wideband beam squint, platform motion은 생략합니다.
  • Not claimed: 실제 전자전 환경의 adaptive jammer, wideband calibrated array, 실시간 beam management를 재현하는 모델은 아닙니다. 배열 신호처리 핵심 개념을 CPU handout으로 확인하는 데 목적이 있습니다.

LCMV 빔포머 — null 배치 원리

LCMV(Linearly Constrained Minimum Variance) 빔포머는 제약 조건을 만족하면서 출력 전력을 최소화하는 가중치 벡터 w를 구합니다. 

최적화 문제:
  minimize   w^H · R · w      (출력 전력 최소화)
  subject to C^H · w = f      (선형 제약)

제약 행렬 C 와 목표 벡터 f:
  C = [a(θ_s),  a(θ_j)]       (원하는 방향, null 방향 스티어링 벡터)
  f = [1,       0      ]       (원하는 방향 응답=1, null 방향 응답=0)

닫힌 형태(closed-form) 해:
  w = R^{-1} · C · (C^H · R^{-1} · C)^{-1} · f

결과:
  w^H · a(θ_s) = 1   → 원하는 신호를 왜곡 없이 통과
  w^H · a(θ_j) = 0   → 재머 방향 응답 = 0 (null steering)

그림 3. LCMV 빔포머 수식. θ_j 추정이 정확할수록 실제 재머 방향 응답이 낮아집니다.

null depth(dB)는 null 방향에서의 빔 응답 전력을 dB로 나타낸 값입니다. 값이 낮을수록(예: -30 dB 이하) 재머가 더 잘 억압됩니다. θ_j 추정이 틀리면 null이 재머를 빗나가 억압 효과가 크게 줄어듭니다.

3. 데이터 생성 코드 Walkthrough — generate_data.py

steering_vector() — 스티어링 벡터 계산

from shared.doa_utils import steering_vector

N_RX = 8          # ULA 안테나 수
D_OVER_LAM = 0.5  # 안테나 간격 / 파장 (λ/2 간격)

# 원하는 신호 방향의 스티어링 벡터
a_s = steering_vector(look_angle_deg, N_RX, D_OVER_LAM)  # (8,) 복소 벡터

# 재머 방향의 스티어링 벡터
a_j = steering_vector(j_angle, N_RX, D_OVER_LAM)         # (8,) 복소 벡터

steering_vector(θ, N, d/λ)는 입사각 θ에 대한 N×1 복소 벡터를 반환합니다. k번째 원소: exp(j·2π·(d/λ)·k·sin(θ)). d/λ = 0.5이면 exp(j·π·k·sin(θ))가 됩니다. sin(θ)의 범위가 [-1, 1]이므로 위상 지연이 [-π·k, +π·k]에 한정됩니다.

generate_sample() — 단일 시나리오 시뮬레이션

def generate_sample(rng, n_jammers, look_angle_deg, jnr_db, snr_db,
                    n_snapshots=100):
    # 재머 각도: look_angle과 최소 10도 이상 분리하여 샘플링
    jammer_angles = _sample_jammer_angles(rng, n_jammers, look_angle_deg,
                                          min_sep=10.0)

    jnr_lin = 10 ** (jnr_db / 10.0)   # JNR을 선형 스케일로 변환
    snr_lin = 10 ** (snr_db / 10.0)   # SNR을 선형 스케일로 변환

    # 수신 신호 행렬 X: (N_RX, n_snapshots)
    X = np.zeros((N_RX, n_snapshots), dtype=np.complex64)

    # 재머 신호 추가
    for j_angle, j_pwr in zip(jammer_angles, jammer_powers):
        a_j = steering_vector(j_angle, N_RX, D_OVER_LAM)  # (8,)
        s_j = np.sqrt(j_pwr/2) * (rng.standard_normal(n_snapshots) +
                                   1j*rng.standard_normal(n_snapshots))
        X += np.outer(a_j, s_j)       # 재머 기여분: a_j * s_j^T

    # 원하는 신호 추가
    s_desired = np.sqrt(snr_lin/2) * (rng.standard_normal(n_snapshots) +
                                       1j*rng.standard_normal(n_snapshots))
    X += np.outer(a_s, s_desired)     # 원하는 신호 기여분

    # AWGN 잡음 추가 (noise power = 1)
    noise = (rng.standard_normal((N_RX, n_snapshots)) +
             1j*rng.standard_normal((N_RX, n_snapshots))) / np.sqrt(2)
    X += noise

    # 샘플 공분산 행렬 추정
    R = (X @ X.conj().T) / n_snapshots  # (8, 8) 복소 행렬

    # Frobenius norm 정규화 (스케일 통일)
    R_norm = R / (np.linalg.norm(R, 'fro') + 1e-10)

    # real/imag 채널 분리 → (2, 8, 8) float32
    cov = np.stack([R_norm.real, R_norm.imag], axis=0)

왜 real/imag을 분리하나?

공분산 행렬 R은 복소수 행렬입니다. PyTorch의 Conv2d는 실수 텐서만 처리하므로, real 채널(ch0)과 imag 채널(ch1)로 나눠 (2, 8, 8) 텐서로 만듭니다. 두 채널이 함께 있어야 방향 정보가 온전히 보존됩니다.

Frobenius norm 정규화란?

Frobenius norm은 행렬 원소 전체의 제곱합의 제곱근입니다: ‖R‖_F = √(Σ|R[i,j]|²). 이 값으로 나누면 JNR 크기에 관계없이 R의 전체 스케일이 통일됩니다. 신경망 학습 안정성을 위해 필수적입니다.

2-재머 케이스 처리

# 재머 수: 1 or 2 (무작위 선택)
n_jammers = int(rng.integers(1, 3))   # 1 또는 2

# 2개일 때: 첫 번째 재머가 더 강함 (1~6 dB 차이)
if n_jammers == 2:
    power_diff = rng.uniform(1.0, 6.0)
    jammer_powers = [jnr_lin, jnr_lin / (10 ** (power_diff / 10.0))]

# 학습 타겟: 항상 첫 번째(강한) 재머 각도
primary_jammer_angle = float(jammer_angles[0])

2-재머 케이스에서 약한 재머는 예측 목표가 아닙니다. 강한 재머가 공분산 행렬에 더 뚜렷이 나타나고, 실제로 억압이 더 시급하기 때문입니다. 모델은 항상 "가장 강한 재머"의 방향을 예측합니다.

"이 파라미터를 바꾸면?" 실험 제안

  • jnr_range=(10.0, 40.0) → (5.0, 15.0): JNR이 낮아지면 재머가 공분산에 덜 뚜렷이 나타납니다. 어느 구간에서 성능이 급락하나요?
  • min_sep=10.0 → 5.0: look angle과 재머가 가까워지면 LCMV null과 원하는 방향이 충돌합니다. 어떤 문제가 생기나요?

4. 모델 코드 Walkthrough — model.py

CovNet 전체 구조

class CovNet(nn.Module):
    def __init__(self):
        super().__init__()

        self.cnn = nn.Sequential(
            # (B, 2, 8, 8) → (B, 16, 8, 8)
            nn.Conv2d(2, 16, kernel_size=3, padding=1),  # padding=1: 공간 크기 유지
            nn.BatchNorm2d(16),
            nn.ReLU(inplace=True),
            # (B, 16, 8, 8) → (B, 32, 8, 8)
            nn.Conv2d(16, 32, kernel_size=3, padding=1),
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            # (B, 32, 8, 8) → (B, 64, 8, 8)
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
        )

        # (B, 64, 8, 8) → (B, 64)  : 공간 차원 평균
        self.gap = nn.AdaptiveAvgPool2d(1)

        # look_angle과 concat 후 회귀
        self.head = nn.Sequential(
            nn.Linear(64 + 1, 256),   # 64(CNN 피처) + 1(look_angle)
            nn.ReLU(inplace=True),
            nn.Linear(256, 128),
            nn.ReLU(inplace=True),
            nn.Linear(128, 1),        # sin(jammer_angle) 출력
        )

8×8 입력에 padding=1인 3×3 Conv를 3번 적용해도 공간 크기(8×8)가 유지됩니다. MaxPool 없이 GAP으로 공간 정보를 요약하는 이유는, 8×8은 이미 작은 크기이므로 추가 다운샘플링이 불필요하기 때문입니다.

forward() — look angle 융합 방식

def forward(self, cov: torch.Tensor, look_angle: torch.Tensor) -> torch.Tensor:
    # cov        : (B, 2, 8, 8)  — real/imag 공분산 행렬
    # look_angle : (B, 1)        — 원하는 신호 방향 [deg]

    feat = self.cnn(cov)         # (B, 64, 8, 8): CNN 피처 추출
    feat = self.gap(feat)        # (B, 64, 1, 1): 공간 평균
    feat = feat.flatten(1)       # (B, 64)       : 벡터화

    # look_angle 정규화: [-90°, +90°] → [-1, 1]
    look_norm = look_angle / 90.0   # (B, 1)

    x = torch.cat([feat, look_norm], dim=1)  # (B, 65): CNN 피처 + look_angle 결합
    return self.head(x)          # (B, 1): sin(jammer_angle) 예측

왜 look_angle을 따로 입력하나?

재머 방향은 절대 각도가 아니라 look angle에 대한 상대적 위치가 중요합니다. 공분산 행렬만 보면 어느 방향이 "원하는 방향"인지 알 수 없으므로, look_angle을 외부에서 명시적으로 알려줍니다. 이 정보가 없으면 모델이 재머 방향과 원하는 방향을 혼동할 수 있습니다.

왜 sin(θ)를 예측하나?

각도 θ(degree)를 직접 예측하면 출력 범위가 [-60°, +60°]로 제한이 없고 최적화가 불안정해질 수 있습니다. sin(θ)는 범위가 [-1, 1]로 자연스럽게 제한되어 학습이 안정적입니다. 예측 후 θ = arcsin(clamp(y, -1, 1)) × 180/π로 각도를 복원합니다.

그림 5. main() 실행 흐름

SmoothL1Loss — 손실 함수 직관적 이해

criterion = nn.SmoothL1Loss()

# SmoothL1 (Huber Loss) 동작:
#   |e| < 1:  loss = 0.5 · e²        ← 오차가 작으면 L2처럼 부드럽게
#   |e| ≥ 1:  loss = |e| - 0.5       ← 오차가 크면 L1처럼 이상값에 강인

# e = sin(θ_pred) - sin(θ_true)

# 예시:
#   sin(θ_pred)=0.50, sin(θ_true)=0.52  → e=-0.02  loss≈0.0002  (잘 예측)
#   sin(θ_pred)=0.50, sin(θ_true)=-0.3  → e=0.80   loss≈0.32    (크게 틀림)
왜 SmoothL1인가? MSE(L2)는 이상값(outlier)에 지나치게 민감합니다. 재머 각도 추정에는 가끔 매우 틀린 예측이 있을 수 있고, 이때 SmoothL1이 학습을 더 안정적으로 유지합니다. sin(θ) 공간에서의 손실이므로, 동일한 sin 오차라도 θ가 ±90° 근방에서는 각도 오차가 더 커집니다.

lcmv_null_depth() — 실제 재머 억압 성능 측정

def lcmv_null_depth(R, look_angle_deg, null_angle_deg, eval_angle_deg):
    a_look = steering_vector(look_angle_deg, N_RX, D_OVER_LAM)  # (8,)
    a_null = steering_vector(null_angle_deg, N_RX, D_OVER_LAM)  # (8,)
    a_eval = steering_vector(eval_angle_deg, N_RX, D_OVER_LAM)  # 실제 재머 방향

    # 제약 행렬: C = [a_look, a_null], 목표: f = [1, 0]
    C = np.stack([a_look, a_null], axis=1)  # (8, 2)
    f = np.array([1.0, 0.0])

    R_inv = np.linalg.inv(R + 1e-8 * np.eye(N_RX))  # 역행렬 (정규화 항 추가)
    RiC = R_inv @ C
    w = RiC @ np.linalg.inv(C.conj().T @ RiC) @ f   # LCMV 가중치 벡터

    # 예측 null을 적용한 뒤 실제 재머 방향 응답 전력 → dB 변환
    null_response = abs(w.conj() @ a_eval) ** 2
    null_depth_db = 10 * np.log10(null_response + 1e-20)
    return null_depth_db  # 낮을수록 좋음 (예: -40 dB)

이 함수는 CovNet이 예측한 각도에 null을 배치한 뒤, 실제 재머 방향에서 응답이 얼마나 줄었는지 측정합니다. 예측 각도 자체에서만 측정하면 LCMV 제약 때문에 틀린 예측도 항상 깊은 null처럼 보이므로 성능 지표가 아닙니다.

MUSIC + LCMV 기준선

def music_lcmv_baseline(split="test", n_eval=500):
    angle_grid = np.linspace(-90, 90, 361)  # 탐색 각도 격자

    for i in range(n_eval):
        R = cov_2ch[0] + 1j * cov_2ch[1]   # real/imag → 복소 공분산

        # MUSIC 스펙트럼 계산 (재머 수 + 원하는 신호 = n_sources)
        P = music_spectrum(R, N_RX, angle_grid, n_sources=n_jam + 1)
        peaks = find_spectrum_peaks(P, angle_grid, n_sources=n_jam + 1)

        # look_angle 근처 피크 제거 → 나머지 최대 피크 = 재머 추정
        jammer_cands = [a for a in peaks if abs(a - look_deg) > 5.0]
        est_jammer = jammer_cands[0] if jammer_cands else angle_grid[np.argmax(P)]

MUSIC(Multiple Signal Classification)은 공분산 행렬의 고유값 분해(eigendecomposition)를 이용한 고해상도 DoA 추정 알고리즘입니다. 신호 부공간(signal subspace)과 잡음 부공간(noise subspace)의 직교성을 이용합니다. 계산량은 CovNet보다 훨씬 크지만, 이 프로젝트에서는 비교 기준선으로 사용합니다.

평가 지표 설명

지표 수식 의미 목표값
Angle MAE (deg) mean(|θ_true - θ_pred|) 재머 각도 평균 절대 오차. 낮을수록 좋음. < 1.5 deg
Angle RMSE (deg) √mean((θ_true - θ_pred)²) 큰 오차에 더 민감한 오차 지표. < 2.5 deg
Within ±2° Acc mean(|error| ≤ 2°) 2도 이내로 맞춘 샘플 비율. > 0.80
True-jammer LCMV Response (dB) 10·log10(|w^H·a(θ_true)|²) 예측 null 적용 후 실제 재머 방향 응답. 낮을수록 강한 억압. 낮을수록 좋음

Null Depth는 최종적으로 가장 중요한 지표입니다. 각도 MAE가 작아도 LCMV null이 재머를 제대로 억압하지 못하면 실용적 가치가 없습니다. 특히 look angle과 jammer angle이 가까운 케이스에서 null depth를 확인하세요.
