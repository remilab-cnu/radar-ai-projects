1. 이 프로젝트는 뭘 하는 건가요?

I/Q 수신기란 무엇인가요?

레이다나 통신 수신기는 수신 신호를 I(In-phase, 동상) 채널과 Q(Quadrature, 직교) 채널로 분리하여 처리합니다. 두 채널을 합치면 s(t) = I(t) + j·Q(t)라는 복소 신호가 되며, 이 복소 표현은 진폭과 위상 정보를 모두 담고 있어 레이다 신호 처리의 핵심입니다.

이상적인 수신기라면 I 채널과 Q 채널이 완벽히 90° 위상 차이를 유지하고, 동일한 이득(gain)을 가져야 합니다. 그러나 실제 아날로그 회로는 이상적이지 않습니다.

I/Q 불균형(Imbalance)이란?

아날로그 수신기 회로의 불완전성으로 세 가지 오류가 발생합니다:

Gain Mismatch

I 채널과 Q 채널의 증폭기(LNA, Mixer) 이득이 미세하게 다릅니다. gain_db로 표현하며 ±0.5 ~ ±3.0 dB 수준입니다.

Phase Mismatch

90° 위상 이동기(phase shifter)가 정확히 90°를 만들지 못합니다. phase_deg로 표현하며 ±1 ~ ±15° 수준입니다.

DC Offset

믹서(Mixer)의 LO(Local Oscillator) 누설로 각 채널에 직류 성분이 더해집니다. dc_i, dc_q로 표현합니다.

왜 문제인가? I/Q 불균형이 있으면 FFT 스펙트럼에서 원하는 신호 성분의 거울상(image spur)이 나타납니다. 레이다 RDM에서 허위 표적(ghost target)처럼 보여 탐지기를 혼란시키고, 도플러 추정 오차가 커집니다. 이를 방치하면 시스템 성능이 심각하게 저하됩니다.

I/Q 불균형 수학 모델

손상된 출력 신호는 다음 수식으로 표현됩니다:

[I/Q Imbalance 순방향 모델]

I_out(t) = I(t) + dc_i                               ← DC 오프셋만 더해짐

Q_out(t) = g · (I(t)·sin(φ) + Q(t)·cos(φ)) + dc_q  ← 이득 + 위상 + DC

여기서:
  g   = 10^(gain_db / 20)   [선형 이득, gain_db는 dB 단위]
  φ   = phase_deg × π / 180 [위상 오차, 라디안 단위]

추정해야 할 파라미터: [gain_db, phase_deg, dc_i, dc_q]  → 4개 숫자

그림 1. I/Q 불균형 물리 모델

이 모델을 역으로 풀면 손상된 신호로부터 원본 신호를 복원할 수 있습니다. 핵심 아이디어: CNN이 파라미터 4개를 먼저 추정하고, 역공식으로 신호를 복원합니다. CNN 자체가 신호를 복원하는 것이 아니라, 물리 모델의 파라미터를 맞추는 것입니다.

이 프로젝트의 입출력 구조

손상된 beat signal  (B, 2, 512)
  ch0: I_out  (실수부)
  ch1: Q_out  (허수부)
         │
         ▼
  ┌─────────────────────┐
  │   IQImbalanceCNN    │  Conv1d × 3 + AdaptiveAvgPool + FC
  │    (~133K params)   │  1D CNN 회귀기(Regressor)
  └─────────────────────┘
         │  (B, 4)
         ▼
  [gain_db, phase_deg, dc_i, dc_q]  ← 4개 파라미터 추정값
         │
         ▼  해석적 역변환 (Analytic Inverse)
  I_rec = I_out - dc_i
  Q_rec = (Q_out - dc_q - g·sin(φ)·I_rec) / (g·cos(φ))
         │
         ▼
  복원된 신호  (B, 2, 512)  ← 원본에 가까워야 함

3. 데이터 생성 코드 Walkthrough — generate_data.py

make_radar() — FMCW 레이다 파라미터 설정

def make_radar() -> FMCWRadar:
    return FMCWRadar(
        fc=77e9,          # 반송파 주파수: 77 GHz (자동차 레이다 대역)
        bw=1e9,           # 스윕 대역폭: 1 GHz
        T_chirp=51.2e-6,  # 처프 주기: 51.2 μs @ 10 MHz → 정확히 512 샘플
        N_chirps=1,       # 단일 처프만 사용 (fast-time 신호)
        fs=10e6,          # 샘플링 주파수: 10 MHz
        N_rx=1,           # 수신 안테나: 1개
    )

이 프로젝트는 단일 처프(single chirp)의 fast-time 신호만 사용합니다. 512 샘플의 복소 beat signal이 하나의 데이터 샘플이 됩니다. Range-Doppler Map을 만들지 않고, I/Q 신호 자체를 직접 학습에 사용한다는 점이 P05와 다릅니다.

apply_iq_imbalance() — 불균형 인가의 핵심

def apply_iq_imbalance(signal_iq, gain_db, phase_deg, dc_i, dc_q):
    I = signal_iq[0]   # 원본 I 채널
    Q = signal_iq[1]   # 원본 Q 채널

    g   = 10.0 ** (gain_db / 20.0)  # dB → 선형 이득
    phi = np.deg2rad(phase_deg)      # 도(degree) → 라디안

    # I/Q 불균형 순방향 모델 적용
    I_out = I + dc_i                                    # I에는 DC 오프셋만
    Q_out = g * (I * np.sin(phi) + Q * np.cos(phi)) + dc_q  # Q에는 이득+위상+DC

    return np.stack([I_out, Q_out], axis=0)  # (2, 512)

gain_db가 양수이면 Q 채널이 I보다 강하고, 음수이면 약합니다. phase_deg가 0이 아니면 I와 Q가 완벽히 직교하지 않아 신호가 일그러집니다. 이 함수가 데이터셋의 "오염 과정"이고, 학습 목표는 이 과정을 역으로 되돌리는 것입니다.

def generate_one_sample(radar, rng, snr_db, gain_range=(0.5, 3.0),
                        phase_range=(1.0, 15.0), dc_range=(-0.05, 0.05)):
    # 1~3개 표적으로 레이다 장면 시뮬레이션
    n_tgt = rng.integers(1, 4)
    ...
    signal = generate_scene(radar, targets, snr_db=snr_db, ...)

    beat = signal[0, 0, :512]   # 단일 처프의 복소 beat signal (512샘플)
    scale = np.sqrt(np.mean(np.abs(beat)**2)) + 1e-30
    beat_norm = beat / scale    # 전력 정규화 (스케일 문제 방지)

    I_clean = beat_norm.real    # 원본 I
    Q_clean = beat_norm.imag    # 원본 Q

    # 불균형 파라미터 무작위 샘플링 (부호도 무작위)
    gain_db   = ±rng.uniform(0.5, 3.0)   # ±0.5 ~ ±3.0 dB
    phase_deg = ±rng.uniform(1.0, 15.0)  # ±1 ~ ±15 deg
    dc_i = rng.uniform(-0.05, 0.05)
    dc_q = rng.uniform(-0.05, 0.05)

    corrupt_iq = apply_iq_imbalance(clean_iq, gain_db, phase_deg, dc_i, dc_q)

    return {
        "x_corrupt": corrupt_iq,          # (2, 512) — 모델 입력
        "y_params": [gain_db, phase_deg, dc_i, dc_q],  # 정답 파라미터
        "y_clean": clean_iq,              # (2, 512) — 복원 목표
    }
전력 정규화를 하는 이유: SNR이 다른 여러 장면에서 beat signal의 절대 크기가 크게 다릅니다. 정규화를 하지 않으면 모델이 DC 오프셋 추정 시 스케일 혼란을 겪습니다. dc_range = (-0.05, 0.05)는 정규화된 신호 기준의 오프셋입니다.

4. 모델 코드 Walkthrough — model.py

IQImbalanceCNN 클래스 구조

class IQImbalanceCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            # Block 1: (B, 2, 512) → (B, 64, 506)
            nn.Conv1d(2, 64, kernel_size=7, padding=0, bias=False),
            nn.BatchNorm1d(64),
            nn.ReLU(inplace=True),

            # Block 2: (B, 64, 506) → (B, 128, 251)
            nn.Conv1d(64, 128, kernel_size=5, stride=2, padding=0, bias=False),
            nn.BatchNorm1d(128),
            nn.ReLU(inplace=True),

            # Block 3: (B, 128, 251) → (B, 128, 124)
            nn.Conv1d(128, 128, kernel_size=5, stride=2, padding=0, bias=False),
            nn.BatchNorm1d(128),
            nn.ReLU(inplace=True),
        )
        self.pool = nn.AdaptiveAvgPool1d(1)  # (B, 128, 1)
        self.regressor = nn.Sequential(
            nn.Flatten(),
            nn.Linear(128, 64),
            nn.ReLU(inplace=True),
            nn.Linear(64, 4),   # 4개 파라미터 출력
        )

    def forward(self, x):  # x: (B, 2, 512)
        x = self.features(x)
        x = self.pool(x)     # (B, 128, 1)
        x = self.regressor(x)
        return x             # (B, 4)

2D CNN이 아닌 1D CNN을 사용하는 이유: 입력이 시간축(512 샘플)을 가진 1D 신호이기 때문입니다. P05의 NeuralCFAR가 2D 패치를 처리했다면, 이 모델은 1D 시계열을 처리합니다.

레이어별 Tensor Shape 변화

블록 레이어 입력 Shape 출력 Shape 역할
입력 — (B, 2, 512) (B, 2, 512) 손상된 I/Q 신호
Block 1 Conv1d(2→64, k=7) + BN + ReLU (B, 2, 512) (B, 64, 506) 넓은 수용장으로 저주파 패턴 검출
Block 2 Conv1d(64→128, k=5, stride=2) + BN + ReLU (B, 64, 506) (B, 128, 251) 특징 확장 + 시간 해상도 절반 축소
Block 3 Conv1d(128→128, k=5, stride=2) + BN + ReLU (B, 128, 251) (B, 128, 124) 고수준 특징 추출 + 추가 축소
Pool AdaptiveAvgPool1d(1) (B, 128, 124) (B, 128, 1) 시간 축 전체를 하나로 요약
FC 1 Flatten + Linear(128→64) + ReLU (B, 128) (B, 64) 비선형 결합
출력 Linear(64→4) (B, 64) (B, 4) [gain_db, phase_deg, dc_i, dc_q]

왜 분류가 아닌 회귀(Regression)인가?

이 모델의 출력은 "표적 있음/없음" 같은 클래스가 아니라 연속적인 실수값 파라미터 4개입니다. gain_db = -1.73, phase_deg = 8.2처럼 정확한 숫자를 맞혀야 합니다. 따라서 손실 함수도 BCELoss 대신 L1/SmoothL1 같은 회귀 손실을 사용합니다.

AdaptiveAvgPool1d(1)의 의미

Block 3 출력 (B, 128, 124)에서 시간축 124를 평균내어 (B, 128, 1)로 만듭니다. 이렇게 하면 입력 신호 길이가 바뀌어도 FC 층의 입력 크기가 항상 128로 고정됩니다. 128개의 채널 각각은 "신호 전체에서 해당 패턴이 얼마나 나타났는가"를 요약합니다.

그림 4. main() 실행 흐름

CombinedLoss — Dual Loss 설계 근거

class CombinedLoss(nn.Module):
    def forward(self, pred_params, true_params, x_corrupt, y_clean):
        # 1) 파라미터 추정 손실 (SmoothL1 + 스케일 정규화)
        scale = [3.0, 15.0, 0.05, 0.05]  # [gain_db, phase_deg, dc_i, dc_q]
        loss_params = SmoothL1Loss(pred_params / scale, true_params / scale)

        # 2) 신호 복원 손실 (L1)
        x_corrected = apply_iq_correction(x_corrupt, pred_params)
        loss_signal = L1Loss(x_corrected, y_clean)

        # 두 손실의 가중 합
        return loss_params + 0.5 * loss_signal   # λ = 0.5

파라미터 스케일 정규화가 필요한 이유

gain_db의 범위는 약 ±3, dc_i의 범위는 ±0.05입니다. 정규화 없이 SmoothL1을 쓰면 gain_db 항이 dc 항보다 60배나 커서 모델이 gain만 잘 맞추고 dc를 무시하게 됩니다. / scale로 나눠주면 각 파라미터가 비슷한 스케일의 기여를 합니다.

신호 복원 손실(λ=0.5)의 역할

파라미터 손실만 최적화하면 4개 숫자가 정확해도 실제 신호 복원이 불안정할 수 있습니다 (예: gain과 phase가 동시에 조금씩 틀리면 복원 오차가 큼). 신호 복원 손실은 "파라미터 추정 정확도"가 아닌 "최종 복원 품질"을 직접 최적화하는 안전망 역할을 합니다.

apply_iq_correction() — 해석적 역변환

def apply_iq_correction(x_corrupt, params):
    gain_db   = params[:, 0:1]   # (B, 1)
    phase_deg = params[:, 1:2]   # (B, 1)
    dc_i      = params[:, 2:3]   # (B, 1)
    dc_q      = params[:, 3:4]   # (B, 1)

    g   = 10 ** (gain_db / 20.0)          # dB → 선형
    phi = phase_deg * (π / 180.0)         # deg → rad

    I_c = x_corrupt[:, 0, :]   # 손상된 I
    Q_c = x_corrupt[:, 1, :]   # 손상된 Q

    # 역공식 (순방향의 역연산, 닫힌 형식으로 유도 가능)
    I_rec = I_c - dc_i                                        # DC 제거
    Q_rec = (Q_c - dc_q - g*sin(phi)*I_rec) / (g*cos(phi))  # 이득+위상 역보정

    return stack([I_rec, Q_rec], dim=1)  # (B, 2, 512)

이 함수는 CNN의 출력(예측 파라미터)을 받아 수학 공식으로 신호를 복원합니다. CNN이 블랙박스로 신호를 직접 바꾸는 것이 아니라, 물리 모델의 파라미터를 추정하고 닫힌 역공식을 적용합니다. 이 접근법을 물리 기반 복원(Physics-based Restoration)이라고 합니다.

gram_schmidt_correction() — Gram-Schmidt 기준선

def gram_schmidt_correction(x_corrupt_np):
    I = x_corrupt_np[:, 0, :]   # (N, 512)
    Q = x_corrupt_np[:, 1, :]

    # I 채널을 기준으로 정규화
    I_norm = I / (sqrt(mean(I**2)) + 1e-30)

    # Q에서 I 성분을 제거 (직교화)
    dot = sum(Q * I_norm) / L          # I와 Q의 내적(상관도)
    Q_orth = Q - dot * I_norm          # Q에서 I 방향 성분 빼기

    # 스케일 복원
    Q_norm = Q_orth / (sqrt(mean(Q_orth**2)) + 1e-30)
    ...

Gram-Schmidt 직교화는 파라미터를 모르는 상태에서 I와 Q를 강제로 직교시키는 방법입니다. gain과 phase 불균형은 어느 정도 보정하지만, DC 오프셋을 제거하지 못합니다. 이것이 CNN 방법과의 핵심 차이입니다.

평가 지표 설명

지표 수식 의미 목표값
Gain MAE mean(|gain_pred - gain_true|) 이득 추정 오차. dB 단위. < 0.3 dB
Phase MAE mean(|phase_pred - phase_true|) 위상 추정 오차. 도(°) 단위. < 1.0 deg
NMSE 개선 10·log10(NMSE_corrupt / NMSE_corrected) 복원 후 신호 오차가 줄어든 정도 (dB). 클수록 좋음. > 8 dB
IRR 개선 IRR_corrected - IRR_corrupt (dB) 스펙트럼 허위 성분이 얼마나 줄었는지. 클수록 좋음. > 20 dB
IRR(Image Rejection Ratio)이란? I/Q 불균형이 있으면 FFT 스펙트럼에서 원하는 신호(양의 주파수)의 거울상(음의 주파수)이 나타납니다. IRR은 원하는 성분 전력 대비 이 거울상 전력의 비율입니다. IRR (dB) = 10·log10(P_desired / P_image). IRR이 높을수록 허위 성분이 적고 수신기 성능이 좋습니다. 실제 레이다 수신기 스펙에 직접 연결되는 지표입니다.

Step 3 — 실험 아이디어

실험 A: DC 오프셋 제거

초급

generate_data.py에서 dc_range=(-0.05, 0.05)를 dc_range=(0.0, 0.0)으로 바꿔서 DC 오프셋 없이 학습해 봅니다.

확인 포인트: DC 오프셋 유무에 따라 두 방법의 차이가 얼마나 달라지나요?

실험 B: 파라미터 범위 확대

중급

gain_range=(0.5, 3.0)을 (0.5, 6.0)으로, phase_range=(1.0, 15.0)을 (1.0, 30.0)으로 확대합니다.

예상 결과: 더 심한 불균형에서 모델이 여전히 파라미터를 정확히 추정할 수 있을까요? MAE가 비례해서 커지나요, 아니면 saturate되나요?

확인 포인트: gain_mae_db와 phase_mae_deg 변화를 비교하세요.

실험 C: lambda_signal 조정

중급

train.py의 LAMBDA_SIGNAL = 0.5를 0.0(파라미터 손실만)과 1.0(신호 손실 비중 증가)으로 바꿔봅니다.

예상 결과: λ=0이면 파라미터 MAE는 줄지만 신호 복원 품질이 불안정해질 수 있습니다. λ가 너무 크면 파라미터 추정보다 신호 복원에 치중해 MAE가 증가할 수 있습니다.

확인 포인트: nmse_improvement_db와 gain_mae_db를 동시에 추적하세요.
