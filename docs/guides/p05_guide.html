1. 이 프로젝트는 뭘 하는 건가요?

Physics contract / Allowed simplification / Not claimed

  • Physics contract: FMCW beat signal에서 range-Doppler FFT를 만들고, CUT 중심 15×15 패치를 추출합니다. Classical CA-CFAR baseline은 정규화 영상이 아니라 저장된 patch_power 선형 전력 cell에 적용합니다.
  • Allowed simplification: 단일 수신기, 제한된 SNR bin, mixed clutter, 15×15 local patch classifier로 CPU handout runtime을 유지합니다.
  • Not claimed: log-domain network input이 CA-CFAR의 통계 domain이라는 주장이나, 이 실험이 production-grade full-frame tracker라는 주장은 하지 않습니다.

레이다 수신기와 탐지 문제

FMCW(Frequency-Modulated Continuous Wave) 레이다는 송신 신호와 수신 신호를 믹싱하여 Beat Signal을 만들고, 이를 2D FFT 처리하면 Range-Doppler Map(RDM)을 얻습니다. RDM의 각 셀(cell)은 특정 거리·속도 조합에서의 반사 에너지를 나타냅니다.

표적이 존재하는 셀에는 주변보다 높은 에너지가 집중되고, 그렇지 않은 셀에는 클러터(clutter)와 잡음(noise)만 있습니다. 탐지기(detector)의 역할은 이 RDM에서 "어디에 표적이 있는가?"를 판단하는 것입니다.

CA-CFAR의 동작 원리와 한계

전통적인 CA-CFAR(Cell-Averaging CFAR)는 원본 선형 전력 RDM에서 Cell Under Test(CUT) 주변의 training cells 평균으로 잡음 세기를 추정하고, 목표 Pfa(오탐률)에 맞는 임계값 T = α × 잡음추정을 계산합니다. CUT의 에너지가 T를 넘으면 "표적 있음"으로 판단합니다.

CA-CFAR의 핵심 가정: training cells의 클러터가 균일(homogeneous)하게 분포한다고 가정합니다. 하지만 실제 환경에서는 클러터 경계, 다중경로, 재밍 등으로 클러터가 불균일합니다. 이 경우 잡음 추정이 틀어지고, 오탐 또는 미탐이 급증합니다.

Neural CFAR: 패치 기반 학습으로 해결

Neural CFAR는 수식 기반 임계값 대신, 작은 CNN이 패치 전체의 형상을 학습하여 탐지 결정을 내립니다. 이 저장소의 baseline은 같은 CUT 주변의 patch_power 선형 전력 15×15 패치에 guard/training-cell CA-CFAR를 적용합니다. 모델은 정규화된 2채널 패치를 입력으로 사용하지만, Pfa를 주장하는 classical baseline은 선형 전력 domain에서만 계산합니다.

이 프로젝트에서 모델의 입출력은 아래와 같습니다: 

RDM (Nd × Nr)
     │
     ▼  CUT 중심으로 15×15 영역 추출
  ┌─────────────────┐
  │  15×15 패치     │  ch0: noise-floor 기준 dB magnitude
  │  (CUT 중심)     │  ch1: 패치 내 min-max 정규화
  │                 │  patch_power: CA-CFAR용 선형 전력
  └─────────────────┘
     │  (2, 15, 15)
     ▼
  ┌──────────┐
  │  CNN     │  3개의 Conv-BN-ReLU 블록
  │ (NeuralCFAR) │  GlobalAvgPool → FC
  └──────────┘
     │  (1,) — raw logit
     ▼
  sigmoid → 탐지 확률 p
     │
  p ≥ 0.5 → 표적 있음 (label = 1)
  p <  0.5 → 표적 없음 (label = 0)

3. 데이터 생성 코드 Walkthrough — generate_data.py

make_radar() — FMCW 레이다 파라미터 설정

def make_radar() -> FMCWRadar:
    return FMCWRadar(
        fc=77e9,      # 반송파 주파수: 77 GHz (자동차 레이다 대역)
        bw=1e9,       # 스윕 대역폭: 1 GHz → 거리 해상도 ≈ 0.15 m
        T_chirp=50e-6,# 처프 주기: 50 μs
        N_chirps=128, # 처프 수: Doppler FFT 크기 = 128 → 속도 해상도 결정
        fs=10e6,      # 샘플링 주파수: 10 MHz → Range FFT 크기 = 500
        N_rx=1,       # 수신 안테나 수: 1
    )

이 함수는 레이다 파라미터 객체를 하나 만듭니다. bw(대역폭)이 클수록 거리 해상도가 좋아지고, N_chirps가 클수록 속도 해상도가 좋아집니다. 시뮬레이터는 이 파라미터를 기반으로 최대 탐지 거리(max_range)와 최대 속도(max_vel)를 자동으로 계산합니다.

extract_patch() — RDM에서 패치 추출

def extract_patch(rdm_mag: np.ndarray, r_bin: int, d_bin: int) -> np.ndarray | None:
    Nd, Nr = rdm_mag.shape      # RDM 크기: (Doppler bins, Range bins)
    if (r_bin < HALF or r_bin >= Nr - HALF or   # 경계 검사
            d_bin < HALF or d_bin >= Nd - HALF):
        return None              # 패치가 RDM을 벗어나면 None
    patch = rdm_mag[
        d_bin - HALF: d_bin + HALF + 1,   # 중심 ± 7 → 15행
        r_bin - HALF: r_bin + HALF + 1    # 중심 ± 7 → 15열
    ]                            # 결과: (15, 15)
    return patch

HALF = 7이므로 CUT를 중심으로 위아래·좌우 각 7칸을 잘라냅니다. RDM 경계 근처의 CUT는 패치가 잘리므로 None을 반환하고 무시합니다.

patch_to_channels() — 2채널 입력 만들기

def patch_to_channels(patch: np.ndarray) -> np.ndarray:
    # ch0: 전역 noise-floor 기준 dB magnitude (clutter_model이 이미 정규화)
    ch0 = patch.astype(np.float32)

    # ch1: 패치 내 [0, 1] min-max 정규화
    p_min, p_max = ch0.min(), ch0.max()
    ch1 = (ch0 - p_min) / (p_max - p_min + 1e-8)   # 분모 0 방지

    return np.stack([ch0, ch1], axis=0)  # (2, 15, 15)

ch0 — 전역 문맥 (Global Context)

noise-floor를 기준으로 한 dB magnitude입니다. 값이 클수록 잡음 대비 신호가 강합니다. CNN이 "이 패치가 절대적으로 얼마나 강한가"를 파악할 수 있습니다.

ch1 — 로컬 대비 (Local Contrast)

패치 내부에서만 min-max 정규화합니다. 절대적 세기와 무관하게 "패치 내에서 중심부가 주변보다 얼마나 두드러지는가"를 부각합니다.

2채널을 함께 쓰는 이유: ch0 하나만으로는 저SNR에서 표적과 강한 클러터가 구별이 어렵습니다. ch1이 패치 내 상대적 구조를 알려주어 서로 보완합니다.

4. 모델 코드 Walkthrough — model.py

NeuralCFAR 클래스 구조

class NeuralCFAR(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            # Block 1
            nn.Conv2d(2, 16, kernel_size=3, padding=0, bias=False),
            nn.BatchNorm2d(16),
            nn.ReLU(inplace=True),
            # Block 2
            nn.Conv2d(16, 32, kernel_size=3, padding=0, bias=False),
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(kernel_size=2, stride=2),
            # Block 3
            nn.Conv2d(32, 64, kernel_size=3, padding=0, bias=False),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(kernel_size=2, stride=2),
        )
        self.gap = nn.AdaptiveAvgPool2d(1)   # Global Average Pooling
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(64, 64),
            nn.ReLU(inplace=True),
            nn.Linear(64, 1),                # raw logit 출력
        )

    def forward(self, x):           # x: (B, 2, 15, 15)
        x = self.features(x)        # feature extraction
        x = self.gap(x)             # (B, 64, 1, 1)
        x = self.classifier(x)      # (B, 1)
        return x

GlobalAvgPool의 의미

Block 3까지 통과한 피처맵은 (B, 64, 1, 1)입니다. AdaptiveAvgPool2d(1)은 남은 공간 차원을 하나의 숫자로 평균냅니다. 결과적으로 64개의 피처 채널 각각이 "패치 전체에 걸친 해당 패턴의 평균 존재감"으로 요약됩니다. 이 64차원 벡터를 FC에 넣어 최종 탐지 확률을 계산합니다.

왜 2채널 입력인가?

ch0(전역 dB 크기)만 있으면 절대적 강도는 알지만 패치 내 구조를 보기 어렵습니다. ch1(로컬 정규화)만 있으면 대비는 알지만 저SNR과 고SNR을 구별할 단서가 없습니다. 두 채널을 함께 주면 모델이 "이 패치는 절대적으로도 강하고, 주변보다도 두드러진다"는 복합 단서를 동시에 활용할 수 있습니다.

그림 3. main() 실행 흐름

BCEWithLogitsLoss — 손실 함수 직관적 이해

criterion = nn.BCEWithLogitsLoss()

# 내부 동작 (개념):
# p = sigmoid(logit)          ← 탐지 확률
# loss = -[y·log(p) + (1-y)·log(1-p)]
#
# 표적(y=1)이고 p=0.99이면:  loss = -log(0.99) ≈ 0.01  (작음: 맞게 예측)
# 표적(y=1)이고 p=0.01이면:  loss = -log(0.01) ≈ 4.60  (큼:  틀리게 예측)
# 비표적(y=0)이고 p=0.01이면: loss = -log(0.99) ≈ 0.01  (작음: 맞게 예측)
핵심 직관: 모델이 정답에 가까운 확률을 출력할수록 loss가 0에 가까워집니다. 틀리게 예측할수록 loss가 커지고, optimizer는 이 loss를 줄이는 방향으로 가중치를 업데이트합니다. WithLogits는 sigmoid를 loss 내부에서 수치적으로 안정되게 계산한다는 의미입니다.

cfar_baseline() — linear-power patch CA-CFAR와 비교

def cfar_baseline(patch_power, y_np, pfa=1e-2):
    g = 1   # guard cells: CUT 바로 옆 1칸 (표적 에너지 누설 방지)
    t = 3   # train cells: guard 바깥 3칸 (잡음 추정에 사용)
    # n_train_cells = (2*(g+t)+1)^2 - (2*g+1)^2 = 9^2 - 3^2 = 81 - 9 = 72
    alpha = n_train_cells * (pfa**(-1/n_train_cells) - 1)  # CFAR 계수
    # 각 선형 전력 패치에서:
    #   center_pow = patch_power[7,7]       ← CUT 전력
    #   noise_est  = mean(training cells)   ← 잡음 전력 추정
    #   T = alpha * noise_est               ← 임계값
    #   탐지 여부 = center_pow > T

패치 내에서 CA-CFAR threshold를 적용합니다. 15×15 패치 중심(7, 7)이 CUT이고, 중심에서 guard(1칸)를 제외한 train(3칸) ring의 선형 전력 평균을 잡음 전력으로 사용합니다. 정규화 log-domain network input은 CNN에만 사용하고, classical Pfa 계산은 선형 전력 patch_power에서 수행합니다.

Pd @ Pfa=1e-2 TP / (TP+FN) at threshold where FP/(FP+TN)=0.01 오탐률 1%일 때의 탐지 확률. 레이다 실용 지표. > 0.80
Pd @ Pfa=1e-3 TP / (TP+FN) at threshold where FP/(FP+TN)=0.001 오탐률 0.1%로 엄격할 때의 탐지 확률. > 0.65
Balanced Acc. (Pd + specificity) / 2 표적/비표적 불균형에 강인한 정확도 지표. > 0.85

Pd(Detection Probability)와 Pfa(False Alarm Probability)는 레이다 시스템 설계의 핵심 지표입니다. 실제 레이다 수신기는 고정 임계값(0.5)이 아니라 Pfa를 목표값으로 설정하고 임계값을 역산합니다. Neural CFAR도 score thresholding으로 동일하게 Pfa를 제어합니다.
